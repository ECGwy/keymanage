# 密码管理器 - 安卓平台部署指南

## ⚠️ 重要前提：PWA 安装需要 HTTPS

安卓 Chrome 仅在**安全上下文（HTTPS 或 localhost）**下才会弹出「添加到主屏幕 / 安装应用」。用普通 HTTP（如 `python -m http.server` 在局域网 IP 上访问）属于非安全上下文，**无法安装 PWA**，只能用于桌面浏览器查看调试。因此安卓部署**推荐使用项目自带的 `https-server.js`**（已内置 HTTPS 与多项安全加固）。

---

## 方案一：PWA 安装（推荐，最简单，需 HTTPS）

### 步骤

1. **启动本地 HTTPS 服务器**

   在项目根目录运行：

   ```bash
   node https-server.js
   ```

   - 首次运行会用 OpenSSL 自动生成自签名证书到 `certs/` 目录（需本机已安装 OpenSSL；若缺失会提示并退出）。
   - 默认监听 `0.0.0.0:8443`。
   - 控制台会打印手机访问地址与提示。

2. **确保手机和电脑在同一 WiFi 下**

3. **手机端安装**

   - 打开手机 Chrome 浏览器，访问 `https://<电脑IP>:8443`（如 `https://192.168.1.5:8443`）
   - 因使用自签名证书，Chrome 会提示「您的连接不是私密连接」→ 点击**「高级」** → **「继续前往（unsafe）」**
   - 点击 Chrome 菜单（⋮）→ **「添加到主屏幕」** 或 **「安装应用」**
   - 安装后桌面生成图标，可离线使用

### 优点

- 无需打包 APK
- 支持离线使用（Service Worker 已配置）
- 部署新版后刷新即更新
- 占用的存储空间极小

### 安全说明（已加固）

- 服务器已做**路径遍历防护**，强制请求限制在 web 根目录内。
- **禁止访问 `certs/` 下的私钥 / 证书文件**（`.pem` 一律 403）。
- 下发 `Content-Security-Policy`、`X-Frame-Options`、`X-Content-Type-Options` 等响应头。
- ⚠️ 自签名证书仅用于本地测试；正式对外分发建议使用受信任证书或走方案二（APK）。

---

## 方案二：仅本地调试（HTTP，不推荐用于安装）

若只需在桌面浏览器快速预览、不需要安卓安装，可用任意静态服务器（但安卓无法安装 PWA）：

```bash
# Python 3
python -m http.server 8080

# Node.js
npx serve .
```

然后用浏览器打开 `http://localhost:8080`（桌面）或 `http://<电脑IP>:8080`（同设备）。

---

## 方案三：打包 APK（原生体验）

如需独立 APK 安装包，使用 Capacitor 将 PWA 打包为安卓应用。

### 环境要求

- Node.js 18+
- Android Studio（用于构建 APK）
- Java JDK 17+
- Android SDK

### 步骤

1. **安装依赖**

   ```bash
   npm install
   ```

2. **同步 Web 资源到安卓项目**

   ```bash
   npx cap sync android
   ```

3. **打开 Android Studio 构建**

   ```bash
   npx cap open android
   ```

   在 Android Studio 中：
   - 连接安卓手机或启动模拟器
   - 点击 **Run**（▶）即可安装调试
   - 如需正式 APK：Build → Build Bundle(s) / APK(s) → Build APK(s)

4. **安装 APK 到手机**

   生成的 APK 路径：`android/app/build/outputs/apk/debug/app-debug.apk`

   发送到手机安装即可。

> 注意：APK 与 PWA 各自使用独立的 localStorage，**数据不互通**；跨端迁移请用「导出 / 导入备份」。

---

## 跨设备数据迁移（重要）

1. 在**旧设备**「设置 → 导出备份」，得到加密的 `password-backup-*.json`。
2. 在**新设备**安装好本应用（PWA 或 APK）后，**首次启动的登录页**会出现「📥 导入已有备份」按钮。
3. 点击选择备份文件 → 页面刷新后输入**原主密码**解锁，数据即完整恢复。

> 迁移的是加密文件，传输过程安全；但主密码需另行记忆，无法从备份文件恢复。

## 文件说明

| 文件 | 说明 |
|------|------|
| `index.html` | 主页面（单页应用） |
| `https-server.js` | 本地 HTTPS 服务器（安卓 PWA 安装用，含安全加固） |
| `sw.js` / `js/sw-register.js` | Service Worker 离线缓存与注册 |
| `manifest.json` | PWA 清单 |
| `capacitor.config.json` | Capacitor 配置 |
| `certs/` | 运行时生成的 TLS 证书（含私钥，**建议 gitignore，勿提交仓库**） |
| `android/` | 安卓原生项目目录（运行 `npx cap add android` 后生成） |

---

## 注意事项

1. **数据存储**：应用使用浏览器 localStorage 加密存储；PWA 与 APK 的数据不互通。
2. **备份**：定期使用设置中的「导出备份」功能保存数据；新设备可通过首次启动页一键导入。
3. **主密码**：遗忘主密码将无法找回数据，请务必牢记（备份文件本身也是加密的）。
4. **证书安全**：`certs/` 目录含 TLS 私钥，请勿提交到代码仓库；如需重新生成，删除 `certs/` 后重启 `https-server.js` 即可。
5. **网络安全**：`https-server.js` 默认监听 `0.0.0.0`，仅在可信局域网内使用；公网暴露需配合受信任证书与反向代理。
