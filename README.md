# 密码管理器（Password Manager）

一个**纯本地、零云依赖**的密码与隐私信息管理工具。所有数据在浏览器端使用 **AES-GCM 256** 加密后存入本机 `localStorage`，主密码只存在于内存中、永不上传。可安装为 PWA / 安卓 App 离线使用。

> 设计原则：**零知识（Zero-Knowledge）**。服务器/开发者无法看到你的任何明文数据——密钥由你的主密码在本地派生，连备份文件本身也是密文。

---

## ✨ 功能特性

- **三类内置条目**
  - 🔑 应用软件密码（账号、密码、网址、备注）
  - 💳 银行卡信息（卡号、持卡人、银行、有效期、取款密码、CVV）
  - 📍 账单/地址信息（姓名、电话、地址、邮箱）
- **分类筛选侧边栏**：全部 / 应用软件密码 / 银行卡 / 账单，带实时计数。
- **搜索**：按标题、账号、备注实时过滤。
- **收藏**：常用条目一键置顶。
- **密码显示/隐藏**：表单与列表中可切换明文。
- **复制**：列表内一键复制账号、密码等字段。
- **导出 / 导入加密备份**：以加密 `.json` 形式迁移数据，跨设备/跨平台可用。
- **主密码修改**：改密后自动用新密钥重新加密所有数据并写回。
- **首次启动「导入已有备份」入口**：新设备无需先建空库，一步导入旧备份。
- **PWA / 离线**：内置 Service Worker（`sw.js`）与 `manifest.json`，可"添加到主屏幕"。
- **手动锁定**：随时清空内存中的主密钥，返回登录页。

## 🔐 安全设计

| 项目 | 实现 |
|------|------|
| 加密算法 | AES-GCM 256（带认证标签，防篡改） |
| 密钥派生 | PBKDF2-SHA256，**600,000** 次迭代（OWASP 2023 建议值） |
| 随机化 | 每次加密使用**随机 12 字节 IV**、**随机 16 字节 salt** |
| 主密码驻留 | 仅存于内存，关闭/锁定即销毁 |
| 存储位置 | 加密后的密文存 `localStorage`（非明文） |
| XSS 防护 | 所有动态文本经 `escapeHtml` 转义 |
| 传输安全 | 内置 HTTPS 服务器（`https-server.js`），PWA 安装必需 |
| 服务器加固 | 路径遍历防护（`../` 拒绝）、`.pem` 私钥禁止访问、CSP / X-Frame-Options 等安全头 |
| 导入校验 | 导入前结构化校验（字段/迭代次数/IV 长度），损坏文件不覆盖现有数据 |
| 浏览器记忆 | 敏感字段关闭 `autocomplete`，防止明文被浏览器保存 |

## 🚀 本地运行

### 方式一：直接打开（最简单）
用浏览器打开 `index.html` 即可使用。注意：`file://` 场景下 PWA"添加到主屏幕"不可用，但功能完整。

### 方式二：HTTPS 服务器（推荐，PWA 安装需 HTTPS）
```bash
# 需要 Node.js 与 openssl（用于生成自签名证书）
node https-server.js
# 浏览器访问 https://localhost:8443
```
- 首次运行会在 `certs/` 自动生成自签名证书（有效期 7 天）。
- 浏览器会提示"不安全"，点"高级 → 继续前往"即可（自签名证书的正常现象）。

### 方式三：静态服务器
```bash
npm start          # npx serve . -l 8080
```

### 安卓构建（Capacitor）
```bash
npm run sync:android   # 同步 web 资源到安卓原生工程
npm run open:android   # 用 Android Studio 打开
```

## 📱 安卓安装为 App（PWA）

安卓 Chrome 只有在 **HTTPS 或 localhost** 安全上下文下才显示"添加到主屏幕"。

1. 电脑上运行 `node https-server.js`。
2. 手机与电脑在**同一局域网**，手机浏览器访问 `https://<电脑IP>:8443`。
3. 手机提示证书风险 →「高级 → 继续前往」。
4. 浏览器菜单 →「安装应用 / 添加到主屏幕」。
5. 此后以独立 App 形式离线运行。

> 详细步骤与排错见 **`安卓部署指南.md`**。

## 🔄 跨设备迁移

数据迁移 = **搬一个加密保险箱，钥匙（主密码）在你脑子里**。

1. 旧设备：「设置 → 导出备份」→ 得到 `password-backup-YYYY-MM-DD.json`（**密文**）。
2. 通过任意方式传输该文件（邮件 / 网盘 / U 盘均可，因文件本身已加密；**切勿把文件路径和主密码一起发送**）。
3. 新设备首次启动登录页 → 点「📥 导入已有备份」→ 选择文件。
4. 页面刷新后切换为解锁页 → 输入**原主密码**解锁。

⚠️ 备份文件可被离线暴力破解，请务必使用**强主密码**；丢失主密码将无法恢复数据。

## ⚠️ 已知限制 / 后续计划

- **自定义分类未实现**：当前仅有内置三类（密码/银行卡/账单），PRD 中"用户自定义分类"尚未开发。
- **闲置自动锁定未实现**：目前仅支持手动锁定，文档中承诺的"无操作自动锁定"暂缺。
- **复制明文残留剪贴板**：复制密码后未自动清空剪贴板，请注意手动清除。
- **无云端同步**：纯本地存储，换设备需手动导出/导入。
